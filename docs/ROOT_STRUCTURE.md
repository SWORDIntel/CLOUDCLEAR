# CloudClear Root Directory Structure

## Essential Files (Keep in Root)

### Build Systems
- `Makefile` - Primary build system (Linux/macOS)
- `CMakeLists.txt` - CMake build system (cross-platform)
- `build.bat` - Windows build script

### Docker Configuration
- `Dockerfile` - Main Docker image
- `Dockerfile.simple` - Simple API Docker image
- `docker-compose.yml` - Full stack deployment
- `docker-compose.simple.yml` - Simple API deployment
- `.dockerignore` - Docker build optimization

### Scripts
- `install.sh` - Installation script
- `cloudclear.sh` - CLI wrapper script
- `test-simple-api.sh` - Simple API test script

### Configuration
- `.env.example` - Environment configuration template
- `.gitignore` - Git ignore rules
- `Caddyfile` - Reverse proxy configuration

### Documentation
- `README.md` - Main project documentation

## Directories

```
CLOUDCLEAR/
├── api/              # Python API servers
├── data/             # Data files and wordlists
├── docker/           # Docker production configs
├── docs/             # All documentation
├── examples/         # Usage examples
├── include/          # C header files
├── scripts/          # Utility scripts
├── src/              # C source code
├── tests/            # Test suites
└── web/              # Web UI
```

## Files NOT in Root

### Build Artifacts (Generated, Not Tracked)
- `cloudclear` - Main binary (generated by `make`)
- `cloudclear-tui` - TUI binary (generated by `make tui`)
- `cloudclear-recon` - Recon binary (generated by `make recon`)
- `*.o`, `*.a`, `*.so` - Compiled objects/libraries

### Build Directories
- `build/` - CMake build directory (git ignored)
- `out/` - Alternative build directory (git ignored)

## Quick Reference

### Build the Project
```bash
make all              # Linux/macOS
build.bat --all       # Windows
```

### Run Docker
```bash
# Simple API (localhost)
docker-compose -f docker-compose.simple.yml up -d

# Full stack
docker-compose up -d
```

### Install
```bash
./install.sh
```

### Run Tests
```bash
# Simple API test
./test-simple-api.sh

# C tests
make test

# Python API tests
cd api && pytest test_server.py
```

## Keeping Root Clean

The root directory should only contain:
1. **Build files** - Makefile, CMakeLists.txt, build.bat
2. **Docker files** - Dockerfiles, docker-compose files
3. **Essential scripts** - install.sh, test scripts
4. **Configuration** - .env.example, .gitignore, etc.
5. **Main README** - README.md only

All other documentation goes in `docs/`.
All source code goes in `src/`, `include/`, or `api/`.
